# Legal readiness

These pages are technical templates, not legal advice. They must be reviewed and approved for the actual jurisdiction, seller and business model before launch. MatMix is not legally ready while placeholders remain or `npm run legal:check` exits non-zero.

## Data inventory

| Data | Collection and requirement | Storage/access | Retention and transfer |
| --- | --- | --- | --- |
| Name and phone | Public checkout; required | `clients`, `orders`; authenticated CRM | Included in DB backups; retention is not approved yet. |
| Address | Checkout currently marks it required | `orders`; CRM | Included in backups. |
| Preferred contact/value | Optional selection; may contain email, Telegram, WhatsApp or MAX identifier | `clients`, `orders`; CRM | Included in backups. |
| Unloading, payment choice, comment | Checkout business fields; comment optional | `orders`; CRM | Included in backups. |
| Order items, totals and weight | Cart plus authoritative server calculation | `orders.items_json`; CRM/order export | Included in backups and historical snapshot. |
| Order events/history | Generated by backend/CRM actions | `order_events`; CRM | Included in backups. |
| Cart and form suggestions | Browser only | `localStorage` on the visitor device | Not a cookie and not sent until checkout submission. |
| CRM session | Created for authenticated CRM use | HttpOnly `matmix.sid` cookie and separate session SQLite DB | Session TTL is configured by the server. Public anonymous browsing does not intentionally save a session. |
| IP/User-Agent | Available to HTTP infrastructure but not stored in order consent columns by this implementation | Reverse-proxy/platform logs, if configured | Actual log provider and retention must be documented. |

No analytics or advertising scripts were found, so no cosmetic cookie banner was added. The current pages reference Google Fonts; deployment/legal review must determine whether to self-host them or list that provider. Backup and operational access are described separately in the production documentation.

## Configuration

Complete these public, non-secret environment values: `LEGAL_BUSINESS_NAME`, `LEGAL_REGISTRATION_ID`, `LEGAL_TAX_ID`, `LEGAL_ADDRESS`, `LEGAL_POSTAL_ADDRESS`, `PUBLIC_PHONE`, `PUBLIC_EMAIL`, `PUBLIC_WORKING_HOURS`, `LEGAL_JURISDICTION`, `LEGAL_PROCESSORS`, `PRIVACY_POLICY_VERSION`, `TERMS_VERSION`, `CUSTOMER_DATA_RETENTION`, `ORDER_DATA_RETENTION`, `LOG_RETENTION`, and `BACKUP_RETENTION`.

Templates deliberately retain placeholders for seller form, effective dates, applicable law, delivery territory/method/cost/time, pickup, unloading, payment methods/timing/documents/currency, returns/categories/time/state/process/documents/exceptions/refunds, liability and force majeure. Do not replace them without owner and legal approval.

The corresponding content variables are `LEGAL_ENTITY_FORM`, `LEGAL_EFFECTIVE_DATE`, `ORDER_CONFIRMATION_TERMS`, `DELIVERY_TERRITORY`, `DELIVERY_METHODS`, `PICKUP_TERMS`, `DELIVERY_COST_TERMS`, `DELIVERY_TIME_TERMS`, `DELIVERY_CHANGE_TERMS`, `UNLOADING_TERMS`, `OVERSIZE_TERMS`, `PAYMENT_METHODS_PUBLIC`, `PAYMENT_TIMING`, `PAYMENT_DOCUMENTS`, `CASHLESS_TERMS`, `REFUND_PROCESS`, `PUBLIC_CURRENCY`, `RETURN_CATEGORIES`, `RETURN_PERIOD_TERMS`, `RETURN_CONDITION`, `RETURN_REQUEST_PROCESS`, `RETURN_DOCUMENTS`, `RETURN_SHIPPING`, `RETURN_EXCEPTIONS`, `REFUND_TIME`, `LIABILITY_TERMS`, and `FORCE_MAJEURE_TERMS`.

Increase a document version whenever wording that a customer accepts changes. The server, not the browser, stores the current privacy and terms versions, consent timestamp, boolean and canonical URLs on the order. It does not store full document HTML, IP or User-Agent. Old orders remain readable and are shown as having no recorded consent.

Run:

```bash
npm run legal:check -- --json
npm run retention:check
npm run test:legal
```

`legal:check` fails on missing config, placeholders, invalid public phone/email, missing pages/footer links or disabled backend enforcement. The read-only retention report shows oldest/newest records and, when retention values are integer days, potential counts older than policy. It never deletes data. Production launch requires a successful legal check after legal approval.

## Manual data requests

1. Verify the requester using an approved procedure.
2. Find the client by normalized phone in CRM/database and list related orders/events.
3. Export only the approved scope over an approved secure channel.
4. Correct data through audited CRM/administrative procedures.
5. Before anonymisation or deletion, obtain a decision on records that must remain for financial, contractual or legal reasons; do not use an automatic one-click deletion.
6. Record the request, decision, operator, date and performed actions without placing unnecessary personal data in logs.

Legal review must confirm jurisdiction, controller/seller identity, lawful basis, required/optional fields, recipients/processors, international transfers, retention, user rights, delivery/payment/return promises, applicable law, and the final effective dates and versions. After deployment, place a test order, verify the consent block in CRM, check canonical legal URLs and rerun the readiness command.
